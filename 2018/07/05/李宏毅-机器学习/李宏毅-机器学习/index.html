“李宏毅机器学习课程”学习笔记。

Part 0-1: Introduction of Machine Learning

机器学习的简要工作原理

机器学习的工作主要可以分为“training（or learning）” 和“Test”两个部分；其中“training”的目的是训练一个模型，而“Test”则是利用训练好的模型进行实际测试，以判断模型的优劣。具体工作模式如下图所示：

“Training”从架构逻辑上来讲是构造一个模型，但从数学角度上来说仅仅是寻找一个函数(function)罢了，通过这个函数能够把输入的数据(图片、声音···)映射成一组向量，寻找合适的函数使得映射出的向量能够反映输入数据的属性，则完成了训练。例如：在二分类问题中：判别输入的图片是猫还是狗，数据进过函数映射后，得到的就是一个二维的向量$y_1,y_2$，其中$y_1$代表输入图片是猫的可能性，$y_2$代表输入图片是狗的可能性，很明显这组向量就能够反映我们的需求(猫狗鉴别)。“Test”则是利用训练好的函数对未知的数据进行测试(或者说应用)，将数据映射为向量，以实现数据属性的分辨。

假设我们有两组数据 $Set_1= \{ a_1, a_2, \ldots, a_M\}$ 和 $Set_2=\{ b_1, b_2, \ldots, b_N\}$ 分别作为训练集和测试集。 首先Training，我们的目的是寻找一个函数，所以首先构造一个function set $F=\{f_1, \ldots, f_N\}$ ($N$有可能趋于无穷大)，每一个元素$f_n$都能对输入数据进行映射，输出一组向量 $y_i=f_n\left( a_i \right)$；然而每一个function映射出来的效果不尽相同，机器必须得能够进行比较不同function的效果；并且由于function太多了，我们不可能一个一个的去比较哪个是最好的function，所以需要一个好的演算法加快寻找最优function的速度，记最优function为$f^*$。最后在利用function$f^*$在测试集上$Set_2$进行训练。

总的来说，构建机器学习模型主要分为三个部分：

  • Define a set of function
  • Goodness of function
  • Pick the best function

机器学习的简要分类

  • 从目标的角度而言，机器学习可以分为“回归模型”、“分类模型”、“结构模型”；
  • 从function的性质来讲，机器学习可以分为“线性模型”和“非线性模型”；
  • 而从对数据集的要求情况下，机器学习又可以分为“监督学习”、“半监督学习”、“转移学习”、“非监督学习”和“强化学习”。就分类问题而言，以上几类问题可以描述为：
    • “监督学习”是指训练的数据都是有标签的，机器知道每一个数据的类别，如：训练的时候，输入一张狗的照片，机器知道这是狗。
    • “半监督学习”中大部分训练数据是没有标签的，机器只知道小部分数据的类别，这些没有标签的数据，机器也能够利用起来（这些没标签的数据都属于需要分别出来的类别中）。
    • “转移学习”中对于需要分类的数据中（测试集），在训练集中只有一小部分关于他们的数据，但是机器还知道大量的不相关数据的类别，机器同时也利用这些数据进行学习，已得到更好的分类效果。如：机器要进行猫猫狗狗分类，但带有标签的数据不多，但是却有很多关于人、花、树等标签的数据，机器同时对这些数据的学习也可以提高分类效果。
    • “非监督学习”中所有训练数据都是没有标签的，更多的是学习一种模式，比如：输入大量的动物图片，试图画出一张新的图片。
    • “强化学习”与监督学习相对比，监督学习是告诉机器怎么做是对的；而强化学习则是告诉机器，做的对还是错，让机器自己去反思为什么会有这样的结果。

part 0-2: Why we need to learn machine learning?

在未来AI技术会取代很多工作，但是新的职业也同样会诞生，例如AI训练师。人工智能在现在依然有很多地方需要人工的力量，例如：

  • 挑选合适的model和loss function
  • 某些模型的最佳化比较难，如：深度学习

ML Lecture 1: Regression - Case Study

在这一章节，将举例演示说明什么是“回归（Regression）”

intro

在生活中，我们经常会遇到回归问题，例如：炒股的时候，我们希望可以通过有关A公司的一些资料，推断出明天A公司股票的点数；或者对于现在比较热门的智能驾驶模型中，汽车需要通过各种传感器给他反馈的信息，推断出一个他下一时刻需要将方向盘打到什么角度；再或者通过用户的购买情况，推断出用户A购买商品B的可能性；等。所谓“回归”，就是利用已有的一些历史数据推测出未来某一变量的值。

Pockmon Test

接下来，我们将以“pokemon”的进化情况为例，利用进化前的pokemon的CP(combat power)值来推测进化后的CP值。如 图2-1所示，他的属性为：

  • 战斗值$x_{cp}$：14
  • 类别$x_s$：Bulbasaur
  • 生命值$x_{hp}$：10
  • 体重$x_w$：11.62
  • 身高$x_h$：0.88

我们的目标是找到一个函数，当将这些属性输入后能够得到进化后的CP值

数据源于：

根据上一章节的介绍，为实现机器学习，我们需要做：1），model:找到一个函数集合 ；2），评估函数的好坏；3）、找一个最好的函数。

Model

我们先取一个简单的Function set（随意取的）：$y=b+w \cdot x_{cp}$，其中$b$和$w$可以取任意值，不同的$b$和$w$就可以构造出不同的函数。比如说：
$$y=2+3\cdot x_{cp} \\
y=1.5+8\cdot x_{cp} \\
\cdots$$
其中$x$就是特征(Featur)。或者考虑一个复杂的模型：
$$y=b + \sum{w_ix_i}$$

Goodness of function

如何评价一个函数是否好呢？

———利用真实的数据集进行比对

假如我们有一些pockmon进化前的的CP值和进化后的CP值的数据$S=\{s_1,s_2,\ldots,s_n\}$，其中$s_i=\{x^i,\hat{y}^i\}$。通过将$x^i$输入到函数里面，得到一个预测数值$y^i=f\left(x^i\right)$，然后比较预测数值$y^i$和实际数值$\hat{y^i}$，就可以分析出该函数的好坏。最常规的做法是利用差值的平方和来表示：
$$L(w,b)=\sum^{10}_{n=1}(\hat{y}^n-y^i)^2$$
这个函数$L(w,b)$被称为损失函数。

Pick the best function

我们的目标是选择一组最好的参数$(w,b)$使得损失函数最小，即所选择的函数与实际的数据最为匹配，用数学的表达方式为：
$$(w,b) = arg \text{ } \mathop {\min }\limits_{w,b} L(w,b)$$

一种很直观的方法是利用梯度下降法。所谓梯度下降法就是利用梯度寻找局部最优解，再通过反复迭代的方式找到全局最优解（不一定能做到）。假设损失函数只有一个未知参数的$w$，$w^* = arg \text{ } \mathop {\min }\limits_{w} L(w)$。

梯度下降法求最优的模型

随机选取一个初始点$w^0$；
计算损失函数$L$在点$w^0$处的梯度$\frac{dL}{dw} \mid_{w=w^0}$，再沿着梯度下降的方向选择一个新的点 $w^1 = w^0 - \eta \cdot \frac{dL}{dw} \mid_{w=w^0}$，其中$\eta$ 被称为学习率；
然后沿着梯度方向选择新的点$w^2 = w^1 - \eta \cdot \frac{dL}{dw} \mid_{w=w^0}$
反复迭代······直到很接近为止。
