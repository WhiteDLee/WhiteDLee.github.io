Private instructions of Git

Posted on 2021-02-21 | In Instructions |
Recording some useful operations of Git.

Basic info.

  1. Download from “https://git-scm.com/“;

Use it by Git Bash

  1. Open “Git->Git Bash”, then a window popped up;

  2. Log in your Github account by

  3. Make one path as the repository that git can organize by

  4. Add a file (e.g. “readme.txt”) into the repository by

  5. Then tell the git which change you did by

Note: we can use “git –help” to find some instructions or “get help -commit” to find some detailed.

Note: Git operations should be excuted in the repository path.

  1. use “git status” to check whether the files in this repository has been changed, for example

  2. Show what changes have been done by “git diff” (from last add to now), for example:

  3. Show modify record by “git log”, which can show “Author”, “Date” and “uploaded commit words”

  4. Back to previous version by “git reset –hard head~1”, where “head~n” means back to last n version.

  5. If you want to back to one special version (even though the version you abandoned), you can use “git reset –hard f32d6”, where “f32d6” is the number of the version.

  6. If you forget the version num, you can use “git reflog” to see all version num

  7. discard changes in working directory by using “git restore”

  8. delet files:

    “rm”: remove any files and can be used anywhere;
    “git rm”: remove the files in the Git workspace and stage, and only be used in Git workspace.
    “git rm –cached”: remove the files in the stage, and only be used in Git workspace.

  9. establish branch by “git checkout -b”, where “-b” means “establish and switch”; or you finish it step by step, “git branch” and “git checkout”;

  1. show all branches and current pointed branch by “git branch”

  2. merge the branch with current branch. (at the same time, deleting the branch by “git merge –no-ff”)

  3. delete branch

  4. use “git stash” to store current working space; use “git stash list” to print all stored working space; use “git stash pop” to restore stored workspace.

  5. use “git cherry-pick” to copy special changes.

    Note: I am not very clear about this commend.

  1. we can use “git log –graph” to see the branch graph.

Note: we can use “git rebase” to comb branch graph log.

  1. tag. Actually, it is adding a point with “tag name” into certain “commit”.

  2. ignore some special types of file, which will not be orgonized by “Git”. The method is downloading relative “.gitignore” file according to your demand in the link or establish your own ignore list (guiding by the link). Then add this file into the repository.

  3. configure alias and build Git server, please see the link

Knowledge of Git

  1. The relationship between workspace and repository and the meaning of command “add” and “commit”, and “restore”. After executing “git add”, “Git” will put your edited files into stage (like cache) from workspace; after excuting “git commit”, “Git” will put your edited files into master from stage, as following; after excuting “git restore”, the file in the stage will substitute the file in the workspace and the file in the master will substitute the file in the stage.

Note: the version storaged in the master by “commit” is not the version in the workspace but in stage.

  1. Branch (useful for teamwork). Using a point to choose which branch you go.
